I would like to create a local web application for drawing Commodore 64 VIC-II multicolour character sets.

Build this as a straightforward TypeScript web app using Vite, HTML canvas, and simple CSS.

Requirements:
- No backend.
- It should run easily on Windows 11 desktop/laptop using normal Node/npm tooling.
- Keep the code clean, modular, and straightforward to understand.
- Avoid clever or over-engineered architecture.
- Store project data locally in the browser.
- Allow export/import as JSON.
- The app should be intuitive and simple to use.
- Include undo/redo support.

The app is specifically for Commodore 64 VIC-II multicolour character mode.

Important C64 graphics rules:

- A C64 character is 8 bytes.
- Each byte represents one row of the character.
- In multicolour character mode, each row is interpreted as four 2-bit pixels.
- Each 2-bit pixel is displayed double-width.
- Therefore, a character is logically 4x8 multicolour pixels, but appears as an 8x8 screen cell.
- The 2-bit values map to colours like this:

  00 = background colour register D021
  01 = multicolour register D022
  10 = multicolour register D023
  11 = per-cell colour from colour RAM

- The per-cell visible colour is limited to C64 colour indexes 0..7.
- Colour RAM values 8..15 enable multicolour character mode for that screen cell.
- Therefore, when storing/exporting colour RAM values for direct use on the C64, store the selected visible per-cell colour as selectedColour + 8.
- The charset binary itself must contain only the 2048 bytes of character bitmap data:
  256 characters * 8 bytes each.
- Per-character or per-tile colour information is editor/project metadata and should be exported separately from the raw charset binary.

Important distinction: character data vs screen/tile colour data:

- Character bitmap data contains only pixel bit-pair values.
- Character bitmap data does not contain C64 colour RAM values.
- On real C64 hardware, colour RAM is per screen character cell, not per character definition.
- The editor may store a default visible colour per character as metadata.
- This default colour is useful for previews and exports.
- When a character is used in a tile or screen map, that placement may eventually have its own colour RAM value.
- For version 1, use each character's default visible colour for tile previews and colour exports.
- Because colour RAM is per screen character cell, a 2x2 tile can have up to four different per-cell colours: one for each character cell.

Main app concept:

The application has two main editing areas:

1. Character editor
2. Tile editor

============================================================
1. Character editor
============================================================

The top part of the screen is the character editor.

Features:

- Show all 256 characters in a selectable character grid.
- When a character is selected, show it enlarged for editing.
- The editor should show a 4x8 logical multicolour pixel grid.
- Each editable pixel can have one of four logical values:

  0 = D021 background
  1 = D022 multicolour 1
  2 = D023 multicolour 2
  3 = per-cell colour

- Let the user select the active drawing value from these four logical colour slots.
- Let the user configure the global C64 colours for:
  - D021 background colour
  - D022 multicolour 1
  - D023 multicolour 2

- Let the user configure a default visible per-character colour for each character.
- This default visible per-character colour must be limited to C64 colour indexes 0..7.
- This default per-character colour is editor metadata and is useful for previews and exports, but it is not part of the raw charset bitmap data.

Character editing behaviour:

- Left click paints the selected logical colour value.
- Right click samples the logical colour value from the clicked pixel.
- Support click-and-drag painting.
- Support undo/redo.
- Support mirror horizontally.
- Support mirror vertically.
- Support clear character.
- Support copy/paste character.

Character preview:

- Provide a logical edit view showing the 4x8 multicolour pixel grid.
- Provide a C64 display preview where each logical multicolour pixel is displayed double-width, producing an 8x8 preview.
- The enlarged editor can use square logical cells for ease of editing, but the preview should show the true C64 double-width appearance.

============================================================
2. Tile editor
============================================================

The bottom part of the screen is the tile editor.

Features:

- Tiles are composed from character indexes.
- The default tile size is 2x2 characters.
- The user can change tile width and height in characters.
- There are 256 tile definitions.
- Each tile cell references one of the 256 characters.
- Tiles may reuse characters.

Tile dimensions:

- A 1x1 character tile is 4x8 logical multicolour pixels and displays as 8x8 normal C64 pixels.
- A 2x2 character tile is 8x16 logical multicolour pixels and displays as 16x16 normal C64 pixels.
- A 2x3 character tile is 8x24 logical multicolour pixels and displays as 16x24 normal C64 pixels.
- In general:
  - logical multicolour pixel width = tileWidthInChars * 4
  - logical multicolour pixel height = tileHeightInChars * 8
  - displayed C64 pixel width = tileWidthInChars * 8
  - displayed C64 pixel height = tileHeightInChars * 8

Tile editor behaviour:

- Show a selectable list/grid of the 256 tiles.
- When a tile is selected, show an enlarged editable tile layout.
- Allow the user to assign character indexes to each cell of the tile.
- For version 1, this can be done by selecting a character from the character grid and then clicking a tile cell to assign that character.
- If the user holds Shift and left-clicks a character cell inside a tile, select that character in the main character editor for convenient editing.
- For preview, each tile cell should use that character's default visible per-character colour.
- Support copy/paste tile.
- Support clear tile.

============================================================
Encoding / decoding
============================================================

Create utility functions for:

- encodeMulticolorRow(pixels)
- decodeMulticolorRow(byteValue)
- encodeCharacter(characterPixels)
- decodeCharacter(characterBytes)
- exportCharsetBinary()
- exportCharacterVisibleColours()
- exportCharacterColourRamBytes()
- exportTilesFlat()
- exportTilesSeparated()
- exportProjectJson()
- importProjectJson()

Encoding a row:

- Each character row has four logical multicolour pixels.
- Each pixel is a 2-bit value.
- The leftmost pixel goes into bits 7..6.
- The next pixel goes into bits 5..4.
- The next pixel goes into bits 3..2.
- The rightmost pixel goes into bits 1..0.

Example:

function encodeMulticolorRow(pixels: [number, number, number, number]): number {
  return (
    ((pixels[0] & 0b11) << 6) |
    ((pixels[1] & 0b11) << 4) |
    ((pixels[2] & 0b11) << 2) |
    ((pixels[3] & 0b11) << 0)
  );
}

function decodeMulticolorRow(byteValue: number): [number, number, number, number] {
  return [
    (byteValue >> 6) & 0b11,
    (byteValue >> 4) & 0b11,
    (byteValue >> 2) & 0b11,
    byteValue & 0b11,
  ];
}

============================================================
Exports
============================================================

Add export options suitable for Oscar64 or other C64 tooling.

Required exports:

------------------------------------------------------------
1. Raw charset binary
------------------------------------------------------------

- Export a .bin file containing exactly 2048 bytes.
- Character 0 first, then character 1, and so on through character 255.
- Each character exports 8 bytes.
- Each byte encodes four 2-bit pixels:
  - leftmost pixel in bits 7..6
  - next pixel in bits 5..4
  - next pixel in bits 3..2
  - rightmost pixel in bits 1..0
- Do not include any file header.
- Do not include colour information in this file.

------------------------------------------------------------
2. Character colour data
------------------------------------------------------------

The editor stores a default visible per-character colour for each of the 256 characters.

This is editor metadata, not part of the raw charset bitmap.

Provide two export options:

A. Visible colour indexes

- Export 256 bytes.
- One byte per character.
- Each value is 0..7.
- This is editor-friendly metadata representing the visible per-cell colour.

B. C64 colour RAM bytes for multicolour mode

- Export 256 bytes.
- One byte per character.
- Each value is visibleColour + 8, therefore 8..15.
- These values can be written directly to C64 colour RAM for screen cells using these characters.
- Make clear in comments/UI that colour RAM is per screen cell.
- This export is a convenience based on each character's default visible colour.

------------------------------------------------------------
3. Tile definitions, flat format
------------------------------------------------------------

- Export tile character indexes in row-major order.
- For example, for 2x2 tiles:

  tile 0 top-left
  tile 0 top-right
  tile 0 bottom-left
  tile 0 bottom-right
  tile 1 top-left
  tile 1 top-right
  tile 1 bottom-left
  tile 1 bottom-right
  and so on.

- This should adapt to different tile widths/heights.
- Export as a raw .bin file containing one byte per tile cell.
- Each byte is a character index from 0..255.

------------------------------------------------------------
4. Tile definitions, separated-plane format
------------------------------------------------------------

- Export one file per tile cell position.
- For 2x2 tiles, export four files:
  - tile_x0_y0.bin = top-left character indexes for all tiles
  - tile_x1_y0.bin = top-right character indexes for all tiles
  - tile_x0_y1.bin = bottom-left character indexes for all tiles
  - tile_x1_y1.bin = bottom-right character indexes for all tiles

- For other tile dimensions, generate one file per x/y tile-cell position.
- Name files clearly:
  - tile_x0_y0.bin
  - tile_x1_y0.bin
  - tile_x0_y1.bin
  - tile_x1_y1.bin
  etc.

- Each separated file should contain 256 bytes:
  one character index for each of the 256 tile definitions.

Export/import the full editor project:
